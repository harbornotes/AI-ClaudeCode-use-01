1. 最初に使うおすすめ指示

編集作業に入る前（ステップ1〜2）に、状況の把握と方針決定を行うための基本プロンプトです。

まず読ませる
ファイルの構造や内容を把握させるための指示です。
このフォルダ内のファイル構成を確認してください。 まだ編集はしないでください。
改善方針だけ出させる
編集前に、どのような修正アプローチをとるか提案させます。
index.htmlをHTML図解として改善する場合の方針を出してください。 まだ編集はしないでください。 改善点を、レイアウト・配色・印刷・可読性に分けて整理してください。
編集前に確認させる
意図しない箇所の書き換えを防ぐための事前確認です。
これから編集する前に、どのファイルのどの部分を変更する予定か説明してください。 まだ編集はしないでください。

2. 実際に依頼できる具体例 ＆ 3. 便利な依頼パターン

目的や修正範囲を明確に絞った、実践的なプロンプトの例です。

印刷用（改ページ不要・濃い色・余白を狭く）
目的： A4印刷時のページ数をできるだけ減らすこと。 条件： ・図やカードがページ途中で分割されても構いません ・break-inside: avoid は原則使わないでください ・page-break-inside: avoid も原則使わないでください ・@media print を追加してください ・印刷時の余白は小さめにしてください ・文字サイズは読みやすさを保ちつつ、過度に大きくしないでください ・文字のカラーも印刷時に読みやすい色に変更してください ・画面表示のデザインはできるだけ変更しないでください
印刷最適化
index.htmlをA4印刷向けに調整してください。 @media printを追加し、背景色が印刷されなくても読めるようにしてください。 図やカードがページ途中で切れにくいようにしてください。
視認性改善
index.htmlを編集してください。 目的は、図解としての視認性を高めることです。 今回は、余白・見出し・カード配置だけ改善してください。 文章内容は変更しないでください。
文字サイズ調整
index.html内の文字サイズを調整してください。 最小フォントサイズは0.95rem以上にしてください。 小さすぎる注釈やラベルがあれば読みやすくしてください。
SVG修正
SVG部分を修正してください。 矢印とテキストが重ならないようにし、各ボックスの間隔を均等にしてください。 HTML全体の構造は変えないでください。
配色改善
配色を落ち着いたビジネス向けにしてください。 背景は白基調、強調色は1〜2色に抑えてください。 文章内容とレイアウト構造は変更しないでください。
GitHub Pages向け
GitHub Pagesで表示したときに崩れにくいようにしてください。 外部CDNへの依存を避け、HTML/CSS/SVGだけで完結させてください。
部分だけ直す (パターン)
「STEP 3」のカードだけ修正してください。 他のカードや全体レイアウトは変更しないでください。
変更前に提案だけ (パターン)
まず改善案を3つ出してください。 まだコードは編集しないでください。
差分説明 (パターン)
今回変更した内容を、変更前と変更後の違いが分かるように説明してください。
元に戻しやすくする (パターン)
大きな変更は避け、最小限の差分で修正してください。

4. おすすめの基本テンプレ

目的・範囲・条件を構造化して伝えるための汎用テンプレートです。これをベースにカスタマイズしてください。

index.htmlを編集してください。 目的： HTML図解として見やすくすること。 今回の修正範囲： 余白・見出し・カード配置のみ。 条件： ・文章内容は変更しない ・Reactは使わない ・HTML/CSS/SVGで完結 ・font-sizeは0.95rem以上 ・A4印刷でも崩れにくくする ・大きく作り直さず、最小限の差分で改善する

5. 避けるべきこと

1. 一括で全部頼む
「最高のHTML図解に全部作り直してください」
処理が遅くなりやすく、意図しない大改造になりやすいため危険です。
2. 修正範囲を指定しない
「いい感じに直して」
どこを変えるかが不明確なので、レイアウトや機能が崩れる原因になります。
3. 何度も全文を貼る
Claude Codeは直接ファイルを読めます。既存コードはチャットに貼らず、
「index.htmlを読んで修正してください」 で十分です。
4. Google Drive同期中に大量編集する
小規模なら問題ありませんが、大きなプロジェクトでは同期競合（コンフリクト）の原因になります。重要な作業はローカルフォルダでの実行を推奨します。
5. 確認せずGitHub公開する
必ずブラウザでの表示プレビューと、印刷プレビュー（Ctrl+P / Cmd+P）を確認してから公開してください。
